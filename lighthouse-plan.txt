PROBLEMAS CRÍTICOS DETECTADOS
-------------------------------------------------------------

1. FORCED REFLOW MASIVO
   - Archivo: _0v4txvb._.js (chunk de la app)
   - Total reflow time: ~10,068 ms
   - Causa: JavaScript consulta propiedades geométricas (offsetWidth, etc.)
     después de invalidar estilos por cambios en el DOM
   - Impacto: Bloquea el hilo principal ~10 segundos

2. MINIFY JAVASCRIPT
   - Peso total transferido: ~730 KiB
   - Ahorro estimado: 262 KiB
   - Chunks más pesados:
     * next_dist_client: 163 KiB (ahorro 73 KiB)
     * react-dom: 181 KiB (ahorro 49 KiB)
     * node_modules bundle: 135 KiB (ahorro 48 KiB)

3. REDUCE UNUSED JAVASCRIPT
   - Peso total: 785 KiB
   - Ahorro estimado: 418 KiB
   - next-devtools: 213 KiB (135 KiB no usado)
   - next_dist_client: 163 KiB (91 KiB no usado)

4. SPEED INDEX MUY ALTO (12.7 s)
   - Indica que el contenido visual tarda en pintarse completamente
   - Relacionado con el bloqueo del hilo principal

5. MAIN-THREAD WORK (43.4 s)
   - Script Evaluation: 16,570 ms
   - Style & Layout: 11,271 ms
   - Other: 14,175 ms

6. BACK/FORWARD CACHE BLOQUEADO
   - Causa: WebSocket detectado en la página
   - Las páginas con WebSocket no entran en bfcache
   - Impacto: Navegaciones de retorno son más lentas

7. RENDER-BLOCKING REQUESTS
   - 3 archivos CSS bloquean el render inicial
   - Ahorro estimado: 50 ms (bajo impacto, pero fácil de resolver)

PLAN DE ACCIÓN PRIORIZADO
-------------------------------------------------------------

FASE 1 - INMEDIATO (Semana 1-2): CRÍTICO
============================================

1.1 ARREGLAR FORCED REFLOW
    Prioridad: CRÍTICO
    Impacto esperado: Reducir TBT de 4s a <1s
    Acción:
    - Identificar en _0v4txvb._.js qué función consulta offsetWidth
    - Separar lecturas y escrituras del DOM (batch reads/writes)
    - Usar requestAnimationFrame para agrupar lecturas
    - Reemplazar offsetWidth/Height por getBoundingClientRect() si es posible
    - Considerar ResizeObserver en vez de consultas sincrónicas

1.2 ELIMINAR NEXT-DEVTOOLS DEL BUNDLE DE PRODUCCIÓN
    Prioridad: CRÍTICO
    Impacto esperado: -213 KiB transferidos, -135 KiB unused
    Acción:
    - Verificar que NEXT_PUBLIC_DEVTOOLS no esté habilitado en build
    - Asegurar que next-devtools no se incluya en producción
    - Revisar next.config.js para deshabilitarlo

1.3 OPTIMIZAR CARGA DE SWIPER
    Prioridad: ALTA
    Impacto esperado: Reducir bundle JS en ~135 KiB
    Acción:
    - Evaluar si swiper completo es necesario
    - Importar solo módulos específicos (ya se hace parcialmente)
    - Considerar dynamic import para SwiperContainer

FASE 2 - CORTO PLAZO (Semana 2-3): ALTO
============================================
